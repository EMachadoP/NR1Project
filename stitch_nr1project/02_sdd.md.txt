# 02_SDD.md

## Visão Técnica Resumida
A solução será construída como uma aplicação web moderna com frontend em Next.js, backend apoiado em Supabase e funções server-side para regras críticas, relatórios e integração com IA. O sistema prioriza simplicidade de implementação, segurança, anonimato e rastreabilidade.

## Arquitetura Sugerida
Arquitetura full-stack web com:
- frontend PWA para respondente
- frontend web administrativo para RH/Admin/Gestores
- backend via API Routes e/ou Edge Functions
- Postgres para persistência
- Storage para PDFs e arquivos
- integração segura com provedor de IA
- geração server-side de PDFs

### Camadas
1. Interface
2. Aplicação / regras de negócio
3. Persistência
4. Integrações externas
5. Auditoria e observabilidade

## Fluxos Principais

### Fluxo 1 — Criação de campanha
1. RH/Admin cria campanha
2. seleciona questionário, período, setor e unidade
3. sistema gera tokens/QR
4. campanha fica disponível para coleta

### Fluxo 2 — Resposta anônima
1. respondente acessa token/QR
2. sistema valida token e prazo
3. exibe termo e questionário
4. usuário responde
5. sistema normaliza risco por item
6. sistema salva resposta
7. sistema gera relatório individual
8. sistema atualiza consolidado da campanha

### Fluxo 3 — Análise RH
1. RH acessa dashboard da campanha
2. visualiza médias por seção
3. visualiza itens críticos
4. solicita relatório analítico
5. sistema aplica regras fixas
6. backend consulta IA com payload estruturado
7. sistema monta plano de ação inicial editável

### Fluxo 4 — Gestão de plano de ação
1. RH revisa ações sugeridas
2. define responsáveis e prazos
3. atualiza status
4. sistema registra audit log

### Fluxo 5 — Indicadores
1. RH registra período
2. informa valores anteriores, atuais e metas
3. sistema calcula variação
4. sistema sugere necessidade de ação

## Autenticação
- respondente anônimo: acesso por token único, single-use e expiração
- RH/Admin/Gestor: Supabase Auth
- RBAC mínimo:
  - Respondente: apenas preenchimento
  - RH/Segurança: campanhas, análises, ações, indicadores
  - Gestor/Diretoria: leitura consolidada e acompanhamento
  - Admin: configuração completa

## Banco
Banco relacional Postgres com:
- integridade referencial
- constraints para status válidos
- índices em campaign_id, questionnaire_id, token, created_at
- policies de acesso por perfil
- anonimização por design no modo anônimo

## Integrações
- OpenAI API para recomendações estruturadas em JSON
- geração de PDF server-side
- gerador de QR
- exportação CSV/Excel

## Deploy
- frontend e server functions em Vercel
- banco e auth em Supabase
- storage em Supabase Storage
- ambientes: dev, staging e production

## Observações Técnicas
- scoring_direction deve ser obrigatório ao publicar questionário
- cálculo de risco deve acontecer no backend para garantir consistência
- relatórios devem ser gerados com templates versionados
- respostas anônimas não devem armazenar identificadores pessoais
- dashboard consolidado deve bloquear filtros que quebrem anonimato
- IA deve operar com payload controlado e guardrails estritos
